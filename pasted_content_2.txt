# Build Native iOS Version of Metro Reader

Using the completed Android implementation as the functional specification, create a fully native iOS application called Metro Reader.

Do NOT create a cross-platform application.

Build specifically for:

- iPhone
- iPad
- iOS 18+
- iPadOS 18+

Use:

- Swift 6
- SwiftUI
- MVVM Architecture
- SwiftData
- Combine
- AVFoundation
- NaturalLanguage Framework
- PDFKit
- EPUB parsing libraries where needed

## Goal

Create the iOS equivalent of the Android Metro Reader app.

All features, workflows, data models, and user experiences should match the Android version unless a platform-native approach would provide a superior experience.

The application should feel like a blend of:

- Classic Windows Phone Metro
- Modern Apple Books
- Premium reading applications

Typography is the primary design element.

Use the same typeface strategy as the Android version.

## Supported Formats

Import and read:

- EPUB
- PDF
- TXT
- HTML
- MOBI
- AZW3

Support:

- Files app
- Drag and drop
- AirDrop
- Share Sheet
- Open In

## Library Features

Store books locally.

Generate:

- Cover art
- Metadata
- Author information
- Reading statistics
- Reading progress

Maintain:

- Bookmarks
- Notes
- Highlights
- Last position

## Reader

Provide a premium reading experience.

Features:

- Dynamic Type support
- Adjustable font size
- Adjustable margins
- Adjustable line spacing
- Dark mode
- AMOLED mode
- Sepia mode
- Landscape support
- Portrait support

Support:

- Table of contents
- Internal links
- Footnotes
- Images

## Audiobook Mode

Convert any imported ebook into an audiobook.

Use:

- AVSpeechSynthesizer
- Apple Neural Voices
- Offline voices

Controls:

- Play
- Pause
- Resume
- Seek
- Previous chapter
- Next chapter

Voice Settings:

- Voice selection
- Speed
- Pitch

## Reading Synchronization

While narration is active:

- Highlight spoken sentence
- Auto-scroll content
- Maintain reading position
- Synchronize audio and text

Provide an experience similar to Apple Books Read Aloud.

## Audio Caching

Optional feature:

Generate audiobook files from chapters.

Support:

- AAC
- M4A

Allow:

- Offline playback
- Cache management
- Storage monitoring

No tracking.

No telemetry.
